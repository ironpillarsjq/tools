## 背景

当前 `calculator.js` 在检测到起始位置晚于结束位置时直接返回错误。需要改为回绕计算模式：从起始读到全本末卷末尾，再从头读到结束位置。

## 目标 / 非目标

**目标：**
- 当 `start > end` 时，自动回绕计算而非报错
- 同书卷起止颠倒也触发回绕
- 结果显示回绕提示文字
- 非回绕正常计算不受影响

**非目标：**
- 不修改圣经数据结构
- 不修改下拉选择器UI
- 不新增用户配置项

## 技术决策

1. **回绕检测条件不变，分支内容改变**
   - 原条件 `startBookIndex > endBookIndex || (同书 && startChapter > endChapter)` 原本返回error，改为返回 `{ success: true, total, wrap: true }`
   - 理由：最小改动，条件分支不变，只改返回值

2. **回绕计算公式**
   ```
   total = 起始卷剩余章数                          // chapters[start] - startChapter + 1
         + 中间到末卷所有章数                       // sum(chapters, start+1 .. 65)
         + 从头到结束卷前的所有章数                  // sum(chapters, 0 .. end-1)
         + 结束卷已过章数                           // endChapter
   ```
   - 同书卷回绕：中间段 = sum(0..65) - chapters[start]（整本去掉起始卷）
   - 跨书卷回绕：逐段累加

3. **UI新增回绕状态样式**
   - `.result.wrap`：暖黄色背景 + 提示文字"已读完全本，从开头继续"
   - 结果行仍显示章数，提示行在下方以较小字号显示

## 风险 / 取舍

- **[无]** 改动范围小，仅两个文件，不影响现有正常流程

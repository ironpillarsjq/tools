## 背景

这是一个从零开始的静态网页应用，无现有代码库。应用是一个单页工具，用于计算用户选择的两个圣经位置（书卷+章节）之间的章数。无后端、无数据库、无外部依赖——所有逻辑在浏览器中运行。

**约束条件：**
- 必须离线可用（所有数据内嵌）
- 和合本圣经结构：66卷书，共1189章
- 单个HTML文件，引用独立的JS数据文件和逻辑文件

## 目标 / 非目标

**目标：**
- 让用户从下拉菜单中分别选择起始和结束的书卷/章节
- 计算并显示两个位置之间（含两端）的总章数
- 根据所选书卷动态更新章节下拉菜单的选项数量
- 校验结束位置不得早于起始位置

**非目标：**
- 经文级别的精确度（仅到章）
- 中英文之外的国际化/本地化
- 用户账号、数据持久化或分享功能
- 服务端渲染或后端API
- 移动端App封装

## 技术决策

1. **三文件结构：index.html、bible-data.js、calculator.js**
   - 理由：将数据（静态圣经元数据）、逻辑（计算）和展示（UI）分离。无需构建步骤即可轻松维护。
   - 备选方案：单HTML文件内联JS。已否决，因为数据和逻辑量较大，独立文件更利于维护。

2. **使用 <select> 下拉菜单选择书卷和章节**
   - 理由：简单、无障碍的原生HTML元素，无需库依赖。选择书卷时动态更改章节下拉菜单的选项。
   - 备选方案：自动补全/文本输入。已否决，因为验证逻辑更复杂且无障碍性较差。

3. **圣经数据使用 JavaScript 数组**
   - 理由：`[{name: "创世记", chapters: 50}, ...]` 简单易读，浏览器可直接使用，无需解析。
   - 备选方案：JSON文件通过fetch加载。已否决，因为静态数据没必要引入异步加载和错误处理的复杂性。

4. **计算方法：按书卷范围累加章数**
   - 同书卷范围（如 创1-创3）：`结束章 - 起始章 + 1`
   - 跨书卷范围：中间书卷全章数 + 起始书卷剩余章数 + 结束书卷已过章数
   - 无效范围（结束早于起始）：显示错误信息

5. **不使用构建工具或框架**
   - 理由：应用足够简单，原生HTML/CSS/JS即可满足需求。引入框架或构建工具只会增加不必要的复杂性。

## 风险 / 取舍

- **[数据准确性]** 硬编码的章数可能存在错误 → 缓解：发布前对照标准来源交叉验证章数；数据集中在单一文件中，便于审计
- **[移动端布局]** 下拉菜单在小屏幕上可能过于紧凑 → 缓解：使用响应式CSS，保证足够的间距和字体大小
- **[浏览器兼容性]** 在较旧浏览器上可能有问题 → 缓解：使用标准ES5/ES6特性；不需要现代API

## 新增需求

### 需求：圣经书卷和章数数据
系统必须包含和合本圣经全部66卷书的章数数据。每条书卷记录必须包含中文书卷名称和总章数。

#### 场景：66卷书全部存在
- **当** 应用加载时
- **则** 圣经数据数组恰好包含66条记录，每条记录都有 `name` 和 `chapters` 属性

#### 场景：章数正确
- **当** 查询诗篇的章数时
- **则** 返回的章数必须为150

### 需求：书卷和章节选择
系统必须提供两组下拉选择器——一组用于起始位置，一组用于结束位置。每组必须包含一个书卷选择器和一个章节选择器。

#### 场景：选择书卷
- **当** 用户从书卷下拉菜单中选中一本书卷时
- **则** 对应的章节下拉菜单必须更新为显示从1到该书卷章数的选项

#### 场景：初始状态
- **当** 页面首次加载时
- **则** 两个书卷下拉菜单必须默认为第一卷书（创世记），两个章节下拉菜单必须默认为第1章

### 需求：章数范围计算
系统必须计算两个选定圣经位置之间（含两端）的总章数。

#### 场景：同书卷、同章节
- **当** 起始位置为创世记1章，结束位置为创世记1章
- **则** 结果必须为1

#### 场景：同书卷、不同章节
- **当** 起始位置为创世记1章，结束位置为创世记3章
- **则** 结果必须为3

#### 场景：不同书卷
- **当** 起始位置为创世记49章，结束位置为出埃及记2章
- **则** 结果必须为 (50 - 49 + 1) + 2 = 4

#### 场景：结束早于起始
- **当** 结束位置早于起始位置（例如 创世记3章 到 创世记1章）
- **则** 系统必须显示错误信息，说明范围无效

### 需求：结果显示
系统必须以人类可读的格式显示计算结果，同时展示两个位置和总章数。

#### 场景：显示有效结果
- **当** 计算出有效的章数范围时
- **则** 显示格式必须为："<起始书卷><起始章>章 到 <结束书卷><结束章>章 = 共<总数>章"

#### 场景：显示错误
- **当** 范围无效（结束早于起始）时
- **则** 显示必须为：明确的错误信息，说明结束位置必须在起始位置之后
